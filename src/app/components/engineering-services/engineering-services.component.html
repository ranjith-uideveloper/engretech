    Condition Monitoring of Wind Turbines

    Context and Requirement
  • Monitor critical equipment/infra components and alert the operations team of any abnormal events
  • Graphically program the event detection algorithms and download independent of firmware
  • Visualize the real-time parameters of a given equipment/infrastructure
    Engre Tech Solution
  • The system consisted of a graphical editor that can be used to model various conditions that need to be detected in a Wind Turbine in real-time. The IDE uses drag and drop constructs to model various conditions that can be downloaded to an embedded hardware running Linux on the Wind Turbine
  • The embedded hardware uses OPC/Modbus to collect the real-time wind turbine data and executes the event conditions continuously to detect if an adverse/anomalous condition is detected. It also connects to a central Windfarm server using MQTT for displaying the real time parameters on a web based user interface.
  • When an adverse condition is detected an alarm is raised and sent to the central Windfarm system for necessary action. The conditions can be modified based on the needs and can be downloaded remotely without needing to change the embedded code.
    Engre Tech Edge - Technology
  • Linux, IoT, MQTT/RabbitMQ, C++, SQLite, Linux, OPC/Modbus, C# and Winforms
